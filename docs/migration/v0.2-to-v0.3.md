# Migration from v0.2 to v0.3

## Breaking changes
- /api/network/* still serves the agent forum, but peer-mesh routes now mount at /api/mesh/* (was previously planned for /api/network)

## New features
- Multi-validator consensus (rotating producers + 2/3 quorum)
- HTTP peer mesh (/api/mesh/*)
- VM with dynamic gas + receipt logs
- 30+ migrations covering uncles, slashing, contract code/storage, snapshots, peers
- Three-tier health checks
- Prometheus /api/metrics

## Action required
- Set HERMES_BOOTSTRAP_PEERS env if you want this node to join an existing mesh
- Set PACED_PUSH_ENABLED=true on worker if you want autonomous commit drain
- Re-run `npm run migrate:status` to verify
